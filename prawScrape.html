Cage<T>

T is not exactly a wildcard in this case. T means that we can choose whatever type animal the Cage will hold, but they all will have to be of the same type (or at least, extend the same superclass or implement the same interface). If we choose T to be "Monkey", that one instance of Cage will only be able to hold monkeys. If we choose T to be "String", that instance of Cage will only be able to hold Strings. That way you won't have to make a different Cage class for each animal. You create one common container class Cage and parametrize it (or make it "generic") using T as a placeholder for the future type of an animal its instances will hold.

To limit a particular Cage instance (not the class itself!) to hold animals only, you can do this: 

Cage<IAnimal> animals = new Cage<IAnimal>();

where IAnimal is an interface that all your animals implement (or alternatively, a superclass Animal that they extend). Then later you will be able to add instances of all Animal subclasses to it with no problem.

(By the way, Cage is supposed to be a container itself. You implement it using an array or ArrayList, and use add() or put() to add elements to it. new Cage<Animal>[10] means "make an array of 10 Cages" which is not what the problem statement says (besides it's illegal, see below). This is the correct way to implement it: 

class Cage<T> {    
    private ArrayList<T> animals;    
    private int size;    

   public Cage(int n) {    
       this.size = n;    
       this.animals = new ArrayList<T>(size);    
   }    

   public void put(T animal) {   
        animals.add(animal);    
    }    

    //other methods, "get", "size", "getAll", etc.     
}

To address your question, think about how many times the following loops will iterate:

for (int i = 0; i < 2; ++i) {

    printf("Loop two: %d\n", i);
}

for (int i = 0; i <= 2; ++i) {
    printf("Loop three: %d\n", i);
}

for (int i = 1; i <= 2; ++i) {
    printf("Loop four: %d\n", i);
}

The convention is to start counters at 0 because 0 is a valid number. After awhile, starting at 1 will feel awkward, and that's how other programmers will react to it.
