Most non-trivial problems can be represented in some object oriented way.

For example, why not create a simple, text based combat system of the likes that you'd usually find in an old school RPG (eg, the original Final Fantasy)? So you've got a party that contains characters and characters have weapons which have stats. You fight an opponent that is also a character and has their own stats.

So you might start out by creating appropriate objects for the characters and their stats. Let's keep things simple and have just two stats: health and attack. When a character attacks an opponent, they deal damage equal to their attack stat, which is subtracted from the target's health. A simplification that can be used early on is having only one-on-one combat where each character alternates turns and can only attack.

Once you have that implemented, implement weapons. Weapons add to existing stats. So a weapon has stats. Further, there can be multiple classes of weapons that provide different damage algorithms. So we might have a superclass Weapon that has a method getDamage(characterStats) and depending on the class of weapon, different stats might be used to determine damage. So a Bow might use a character's speed stat while a Sword uses the strength stat (etc).

Using your imagination, you could also implement abilities (eg, magical attacks).

There's no single way to write the code for this, but it'll provide practice for writing OO code for a problem in which objects are the natural way to represent things (eg, a character, a weapon, and stats can be objects).
